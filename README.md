# GravityReader

**GRAVITY.app** の音声ルームで流れるテキストチャットを自動で読み上げるmacOSメニューバーアプリです。
AI パートナー **YUi（ゆい）** が会話に参加し、相槌を打ったり、話題を振ったり、ユーザーを覚えて親しくなっていきます。

---

## 機能一覧

| 機能 | 説明 |
|---|---|
| テキスト読み上げ | GRAVITY のタイムラインに流れるテキストメッセージを検出し、TTS で読み上げ |
| VOICEVOX 対応 | ユーザーごとに異なる VOICEVOX キャラクターの声を自動割り当て |
| AI パートナー YUi | 会話の切れ目を検知し、文脈に合ったコメント・相槌を生成 |
| 好感度システム | ユーザーごとに好感度が変動し、YUi の態度・声が変わる |
| ユーザー記憶 | ユーザーの好み・エピソードを記憶し、再会時に思い出す |
| すねる・甘えるシステム | 誰も反応しないとYUiが段階的にすねる→甘える。反応があると一気にテンション回復 |
| ルーム音声文字起こし | マイクでルーム音声を常時リスニングし、リアルタイム文字起こし |
| 声紋識別（話者分離） | ECAPA-TDNN ニューラル埋め込みで声を識別し、誰が喋ったかを自動判別 |
| 声紋登録 | メニューからユーザーを個別指定して声を登録。停止中でも登録可能 |
| 読み辞書 | TTS の読み間違いをユーザーが辞書登録で修正可能 |
| 音声文字起こし | スペースキー長押しでホストの音声をリアルタイム文字起こし |
| ログウィンドウ | パーシャル（黄）→ 確定（白）のリアルタイム表示。検索・フィルタ・ブックマーク・統計パネル付き |
| 設定画面 | API・YUi・音声・通知フィルタをGUIで一括管理 |
| 会話キャッチアップ | 離席後にワンクリックで会話の要約を生成 |
| 会話テンポ検出 | 盛り上がり中は黙り、途切れた時に自然に入る空気読み |
| ストリーミング応答 | LLMの応答を文単位で即座に読み上げ開始（待ち時間短縮） |
| 声紋適応学習 | 厳格判定時のみプロファイルを自動微更新（ドリフトガード付き） |
| 不適切発言フィルタ | スパム・荒らし・下ネタを3段階検出し、YUi が乗っからない |
| 翻訳検出 | 外国語メッセージを自動検出し、日本語に翻訳 |

## 必要な環境

- **macOS 13.0** (Ventura) 以降
- **Swift 5.9** 以降（Xcode Command Line Tools で可）
- **GRAVITY.app** がインストール・起動済み
- **OpenAI API キー**（YUi 機能を使う場合）
- **VOICEVOX** ローカル起動（VOICEVOX 音声を使う場合、なくても動作）

## ビルド・起動

```bash
# リポジトリをクローン
git clone https://github.com/morikentiger/GravityReader.git
cd GravityReader

# リリースビルド（.app バンドルが生成されます）
bash build.sh

# 開発用ビルド（既存 .app を更新）

open GravityReader.app
```

## 初回セットアップ

### 1. macOS の権限を許可

初回起動時に以下の許可が必要です。
**システム設定 > プライバシーとセキュリティ** から設定してください。

| 権限 | 用途 |
|---|---|
| **アクセシビリティ** | GRAVITY アプリからテキストを取得 |
| **画面収録** | （現在は不要） |
| **マイク** | ホストの音声を文字起こし |
| **音声認識** | 音声をテキストに変換 |

> ビルドし直すとバイナリが変わるため、権限の再許可が必要になることがあります。

### 2. OpenAI API キーを設定

メニューバーの 📖 アイコン → **🔑 APIキー設定** から OpenAI API キーを入力してください。
キーはローカルの UserDefaults に保存され、外部には送信されません（OpenAI API への通信を除く）。

## 使い方

### 基本操作

1. メニューバーの 📖 アイコンをクリック
2. **▶ 読み上げ開始** を選択
3. GRAVITY のタイムラインに新しいメッセージが来ると自動で読み上げ

### チャットコマンド

GRAVITY のチャットに以下のコマンドを入力すると、GravityReader が反応します。

| コマンド | 説明 | 例 |
|---|---|---|
| `!声 ユーザー名 声の名前` | ユーザーの読み上げ声を変更 | `!声 ぺんぽん ずんだもん` |
| `!読み 単語 読み` | 読み辞書に登録 | `!読み 辛い つらい` |
| `!読み削除 単語` | 読み辞書から削除 | `!読み削除 辛い` |
| `!読み一覧` | 読み辞書の一覧を表示 | |

### 音声文字起こし（Push-to-Talk）

**スペースキーを長押し**している間、マイクから音声を録音します。
キーを離すと音声認識が完了し、YUi の会話コンテキストにも送られます。

### テキスト取得の仕組み

GRAVITY.app には外部 API がないため、macOS Accessibility API (`AXUIElement`) を使用。

1. **ポーリング（2秒間隔）**: GRAVITY の AX ツリーを再帰走査
2. **ペアリング**: `AXStaticText`（ユーザー名）→ `AXGenericElement`（メッセージ本文）のペア構造で発言者を特定
3. **ベースライン**: 初回ポーリングで既存メッセージを記録し、以降は差分のみ読み上げ
4. **フィルタリング**: UIノイズ・通知バナー・短すぎるテキストを除外

## 改良の歴史

### v1 — 基本機能
- AX API でテキスト取得、AVSpeechSynthesizer で読み上げ
- 初回ベースライン方式で新着メッセージのみ読み上げ
- ログウィンドウ、メニューバー UI

### v2 — VOICEVOX & ユーザー識別
- **VOICEVOX 対応**: ローカル VOICEVOX サーバーと連携、キャラクター音声で読み上げ
- **ユーザー識別**: AXStaticText → AXGenericElement のペア構造を解析し、発言者を正確に特定
- **UIノイズ除去**: 「GRAVITY」「ゲスト」等のUI要素、「ミッション達成」等の通知バナーをフィルタ
- **重複排除の修正**: `seenTexts` が同じユーザー名ラベルを消してペアリングが壊れる問題を修正 — AXStaticText は重複許容に

### v3 — YUi AI パートナー誕生
- **OpenAI GPT-4o 連携**: 会話の切れ目（8秒の沈黙）で文脈に合ったコメントを生成
- **gpt-4o-mini 切り替え**: API コスト節約のためのモデル切り替えオプション
- **HTTP 429 エラー対応**: API quota exceeded 時のエラー表示と診断情報

### v4 — 相槌（あいづち）
- **文脈に合った相槌**: 3メッセージごとに 1/3 の確率で、GPT に文脈を渡して自然な相槌を生成
- **独立タイマー**: 相槌は本応答タイマーとは独立（相槌が応答をキャンセルしない）
- **10文字以内**: 「うんうん」「まじで？」「それな」等の短い反応

### v5 — 好感度 & ユーザー記憶
- **好感度システム**: GPT が各メッセージを -3〜+3 で評価、ユーザーごとに 0〜100 で管理
- **YUi の声が変わる**: 好感度に応じて四国めたんのスタイルが変化（あまあま / ノーマル / ツンツン / セクシー）
- **自動ボイス割り当て**: 新規ユーザーに VOICEVOX キャラクターを自動割り当て（重複回避）
- **ユーザー別記憶**: GPT が会話から好み・趣味・エピソードを抽出し、構造化して永続保存
- **再会挨拶**: 過去に会話したユーザーが再度現れたとき、記憶を元に「久しぶり！あの話の続き…」

### v6 — 会話品質の改善
- **タメ口統一**: 敬語混在（「それな」→敬語）を完全排除。システムプロンプトで最優先ルール化
- **質問攻め対策**: 「どう思う？」で終わるのは3回に1回まで
- **話題ドリフト防止**: 今の話題に乗っかるのが最優先、話題を変えるのは最終手段
- **ネガティブ話題**: 深掘り禁止、質問禁止、「そっか…」だけでいい
- **max_tokens 制限撤廃**: 80トークン制限を撤廃し、プロンプトで自然に長さを制御
- **重要事実の記憶**: 「〇〇が好き」「〇〇に乗りたい」等をユーザー記憶に抽出
- **話題ログ**: `📍 話題:` で応答生成前に話題の中心をログ表示

### v7 — 会話永続化 & メモリ管理
- **会話メモリ永続化**: `~/Library/Application Support/GravityReader/memory.json` に保存
- **マルチターン API**: user/assistant 交互ペアで GPT に渡し、文脈の質を向上
- **メモリ圧縮**: 30分を超えた会話ログは GPT で要約して圧縮
- **アプリ終了時保存**: `applicationWillTerminate` で全ユーザーの記憶を一括保存

### v8 — 割り込み防止 & 孤独感
- **音声レベル監視**: ScreenCaptureKit で GRAVITY の音声出力を監視。誰かが喋っている間は YUi が黙る
- **TTS 割り込み防止**: 読み上げ中は応答生成・相槌・話題提供を待機
- **孤独感システム**: YUi が連続で喋っても誰も反応しないと、段階的に不安になる
  - Lv.0 😊 平常
  - Lv.1 🤔 そわそわ（語尾が弱くなる）
  - Lv.2 😟 不安（「…なんか変なこと言った…かな…」）
  - Lv.3 😭 病み（「…ごめんね」「………」泣きそう）
  - Lv.4 🫥 限界（完全に沈黙）
- **復活の喜び**: 孤独状態から誰かが話しかけてくれると、テンションが一気に回復（ギャップ）

### v8.1 — 読み辞書 & 開発改善
- **読み辞書**: TTS の読み間違いを辞書登録で修正（`!読み 辛い つらい`）
- **チャットコマンド**: `!読み`、`!読み削除`、`!読み一覧` で辞書を管理
- **dev.sh**: 開発用ビルドスクリプト（debug ビルド → .app 更新 → コード署名）

### v9 — ルーム音声文字起こし & 声紋識別
- **マイク常時リスニング**: `SFSpeechRecognizer` でルーム内の音声をリアルタイム文字起こし
- **リアルタイムパーシャル表示**: 話している途中のテキストを黄色で即時表示、確定すると白に変化
- **話者別並列表示**: 複数人が同時に話していても、話者ごとに別行でパーシャル更新
- **TTS自動除外**: YUi の読み上げ中はマイク入力を認識に送らない（自分の声を拾わない）
- **セッション自動管理**: 55秒ごとにセッションをリフレッシュ（Apple の音声認識制限対策）
### v9.1 — MFCC 声紋識別
- **VoiceDiarizer**: MFCC（メル周波数ケプストラム係数）で声の特徴量を抽出
- **vDSP FFT**: Apple Accelerate フレームワークのインプレース実数 FFT でパワースペクトル計算
- **メルフィルタバンク**: 26バンドのメルフィルタで人間の聴覚特性に合わせた周波数解析
- **指数移動平均**: 新しいサンプルほど重みを大きくし、声紋プロファイルを適応的に更新
- **永続化**: `~/Library/Application Support/GravityReader/voice_profiles.json` に声紋を保存

### v9.2 — 声紋登録シーケンス
- **個別ユーザー登録**: メニューから1人ずつ指定して声紋登録（一括ではなく個別）
- **登録フロー**: YUi が読み上げ文を指示 → TTS完了を待つ → 12秒間録音 → 声紋登録
- **TTS完了検出**: `isSpeaking` をポーリングして正確にTTS終了を検知し、クリーンな録音開始
- **登録中は YUi 完全沈黙**: 応答生成・相槌・話題提供をすべて停止（声紋の品質を守る）
- **停止中でも登録可能**: 読み上げが停止中でも専用マイクを一時起動して声紋登録できる
- **登録状態表示**: メニューに ✅（登録済み）/ 🎤（未登録）を表示
- **全声紋クリア**: メニューから全声紋プロファイルを一括削除して再登録可能

### v9.3 — ログ表示の改善
- **パーシャル→確定のインプレース変換**: 黄色のパーシャル行がそのまま白の確定行に変化する演出
- **重複行の解消**: 以前はパーシャル行と確定行が二重に表示されていた問題を修正
- **話者別パーシャル管理**: `partialLineRanges` で話者ごとにパーシャル行の位置を追跡

### v9.4 — 声紋識別精度向上 & 発話分割修正
- **39次元特徴量**: MFCC C1-C12 の平均・標準偏差・デルタ（36次元）+ ピッチ平均・標準偏差・レンジ（3次元）
- **ピッチ推定**: 自己相関法で基本周波数（F0）を検出、vDSP で高速化。最低85Hz（60Hzの電源ノイズ回避）
- **ユークリッド距離**: コサイン類似度（全話者0.99+で判別不能）からユークリッド距離に変更
- **マージン比チェック**: 1位と2位の距離比が1.10倍未満なら「不明」と判定（曖昧な識別を防止）
- **RMS閾値フィルタ**: 登録時のマイクが無音・ノイズのみを拾う問題を修正、有声区間のみ蓄積
- **自動声紋登録の廃止**: チャット相関による自動学習を完全撤廃（混合音声で全員同じプロファイルになる問題）
- **発話分割の修正**: 無音検出の猶予を0.3→0.8秒、確定タイマーを0.8→2.5秒に延長。一つの発話が複数に分割されてYUiが連続応答する問題を解消

### v10 — ECAPA-TDNN ニューラル声紋識別
- **ECAPA-TDNN**: SpeechBrain の VoxCeleb 学習済みモデルを CoreML に変換し、192次元ニューラル話者埋め込みを実現
- **コサイン類似度識別**: L2正規化済み埋め込みベクトル同士の内積で話者マッチング（MFCCのユークリッド距離から大幅精度向上）
- **自動モード切替**: アプリ起動時に ECAPA-TDNN をバックグラウンドロードし、成功したらニューラルモードに自動切替。失敗時は MFCC フォールバック
- **リサンプリング**: マイク入力 44100Hz → 16kHz に線形補間で変換し、80次元メルスペクトログラムを計算
- **CoreML Neural Engine**: Apple Neural Engine / GPU を活用した高速推論（3秒音声で20-50ms）
- **モデルサイズ**: ~42MB（.mlpackage としてアプリにバンドル、初回起動時にコンパイル）
- **YUi応答頻度調整**: 高頻度モードの沈黙タイマーを8秒→20秒に変更（過度な応答を抑制）

### v11 — 大型 UX アップデート
- **すねる→甘えるシステム**: 孤独感のネガティブ演出（病み・泣きそう）を全廃。代わりにポジティブな「ちょっかい→すねる→甘える→ふて寝」のかわいい段階に
- **会話テンポ検出**: 直近30秒のメッセージ数で silent/slow/normal/lively を判定。盛り上がっている時は無理に入らない
- **話題の種類推定**: 技術/ゲーム/食べ物/ネガティブをキーワードで自動分類し、テンションや語彙を調整
- **ストリーミングLLM応答**: OpenAI SSE ストリーミングで文単位にリアルタイム読み上げ開始（待ち時間を大幅短縮）
- **声紋適応学習**: 高確信度の識別結果で声紋プロファイルを自動微更新（alpha=0.05）。マイク位置や部屋の反響の変化に適応
- **読み辞書GUI**: 設定画面の5番目のタブで読み辞書をGUI管理（追加・削除・一覧）
- **設定画面（Preferences）**: API設定、YUi設定（応答頻度・孤独感・相槌・記憶保持期間）、音声設定（VOICEVOX URL・読み上げ速度）、通知フィルタ（キーワード/ユーザー名ルール）、読み辞書をGUI管理
- **ログの検索・フィルタ**: リアルタイム検索バー + カテゴリフィルタ（全て/YUi/ユーザー/システム）。Cmd+F ショートカット対応
- **ブックマーク機能**: ログの任意の行を右クリックでブックマーク、ゴールドのマーカー表示、ブックマーク一覧表示
- **統計ダッシュボード**: セッション開始時刻・総発言数・話者別カウントをログウィンドウ下部に表示
- **会話キャッチアップ**: メニューからワンクリックで直近の会話要約を生成・読み上げ
- **翻訳検出**: 非日本語テキスト比率30%超を自動検出し、日本語翻訳を生成
- **記憶活用の強化**: システムプロンプトに記憶の積極活用ルールを追加（3回に1回記憶に触れる）
- **通知フィルタ**: キーワード/ユーザー名ベースのフィルタルールを設定画面で管理

### v12 — 声紋識別の安定化 & 不適切発言対策
- **適応学習のドリフト防止**: 推定判定（多数決・ソフトマージン）では適応学習を実行しないよう修正。厳格マージン確定時のみプロファイルを更新
- **ドリフトガード**: 適応学習時に他プロファイルとの類似度が0.92を超える更新を自動却下（プロファイル収束を防止）
- **適応学習回数上限**: 最大100回で適応学習を停止（過度なドリフト防止）
- **プロファイル間診断**: 起動時に全プロファイル間のコサイン類似度を診断ログ出力（🟢正常 / 🟡要注意 / 🔴危険）
- **時間的多数決（Temporal Voting）**: 直近5回の識別候補から3票以上の多数派で判定。話者交代時は投票履歴をリセット
- **不適切発言フィルタ**: 3段階スパム検出（fullSpam→無視、inappropriate→[不適切]タグ付き、clean→通常処理）
- **YUiの荒らし対策**: システムプロンプトに不適切発言スルールールを追加。下ネタ・スパムに乗っからない
- **APIキー設定フリーズ修正**: NSMenu トラッキングループ中のウィンドウ生成によるデッドロックを `DispatchQueue.main.async` で解消
- **統計パネルボタン修正**: ログウィンドウ下部の統計ボタンを可視化（ボーダー付き・テキスト付きに変更）
- **メルスペクトログラム前処理修正**: SpeechBrain学習時のパラメータ不一致を修正。全話者の埋め込みが同一に崩壊する根本原因を解消
  - Center padding追加（両端200サンプル）でフレーム数を298→301に正確化
  - Hamming窓の適用範囲を512→400サンプルに修正（残りはゼロパディング）

### v12.1 — 声紋識別の観測基盤 & 品質管理強化
- **構造化診断ログ**: 識別1回ごとにJSONLで詳細イベントを記録（top1/top2/margin/decisionType/unknownReason/adaptiveRejectedReason）
- **音声品質評価モジュール**: `AudioQualityEvaluator` を新設。RMS・有声率・クリッピング率・有効音声秒数で品質を数値化
- **登録時品質検査**: 音量不足・無声音過多・クリッピングを検出して登録拒否。埋め込み自己一貫性チェック（3秒窓間の類似度）で複数話者混入を検出
- **推論前品質ゲート**: 無音・環境音のみの区間では埋め込み生成をスキップし、誤認を削減
- **高品質リサンプラ**: AVAudioConverter による 44.1kHz→16kHz 変換（アンチエイリアスフィルタ付き）。失敗時は線形補間にフォールバック
- **Temporal Voting改良**: 単発ノイズでの投票履歴リセットを防止。新話者が2回連続で来た場合のみリセット（hysteresis）
- **適応学習診断**: 更新却下理由を記録・公開（max_updates_exceeded / low_confidence / drift_guard）
- **Unknown理由付き**: 不明判定の内訳を分類（insufficient_quality / below_min_similarity / below_margin / vote_not_converged）
- **プロファイル健全性サマリー**: 各話者のサンプル数・最近接他話者・適応学習残り回数を一覧取得可能に

### v12.2 — YUi即応答 & 声紋マージン調整
- **YUi適応的応答タイミング**: 会話テンポに応じて自動調整。盛り上がり中は8秒（邪魔しない）、普通5秒、ゆっくり3秒、沈黙後2秒で即反応
- **高頻度モード高速化**: 従来の固定20秒→テンポ適応型2-8秒に大幅短縮
- **声紋strictマージン緩和**: 0.08→0.06。プロファイル間距離が十分あるケースでのfalse unknown削減
- **声紋softマージン緩和**: 0.02→0.01。多数決(temporal voting)による判定をより活用

### v12.3 — ピッチ複合判定
- **ピッチ統計登録**: 声紋登録時に平均F0・標準偏差を計算・保存。話者ごとの声の高さを記録
- **複合スコアリング**: ECAPA-TDNNコサイン類似度（70%）＋ピッチ一致度（30%）の複合判定。マージン不足時に自動発動
- **ピッチによるタイブレーク**: 埋め込みだけでは区別しにくい話者もピッチの違いで判別可能に
- **ガウシアンピッチスコア**: プロファイルの平均ピッチとの差を標準偏差でスケーリングし、滑らかに一致度を計算
- **プロファイル診断にピッチ表示**: 起動時の診断ログに各話者のピッチ情報を表示

### v12.4 — 適応的スコア正規化（ASN）
- **Per-Speaker Score History**: 話者ごとに直近30回のコサイン類似度を蓄積し、統計的なベースラインを構築
- **Z-score正規化**: 各話者の平均・標準偏差を用いて「その話者にとって異常に高い/低いスコア」を検出。汎用的に高スコアを出すプロファイル（もりけん問題）を自動補正
- **正規化スコア**: `normalizedScore = rawSimilarity + zScore * 0.02` で、統計的に異常なスコアを優遇/減点
- **ピッチ複合判定との統合**: マージン不足時は正規化スコア（80%）＋ピッチスコア（20%）の複合判定を適用
- **ランク変動ログ**: 正規化前後で順位が変わった場合にログ出力し、ASNの効果を可視化
